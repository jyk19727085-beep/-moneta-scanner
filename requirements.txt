**2. GitHub 업로드**
GitHub에 새로운 퍼블릭(Public) 또는 프라이빗(Private) 저장소를 만드시고, `app.py`와 `requirements.txt` 두 개의 파일을 커밋(Commit)하여 업로드합니다.

**3. Streamlit Cloud 배포**
1. [share.streamlit.io](https://share.streamlit.io/)에 접속하여 GitHub 계정으로 로그인합니다.
2. **"New app"** 버튼을 클릭합니다.
3. 방금 만든 GitHub 저장소와 브랜치(보통 main)를 선택하고, Main file path에 `app.py`를 입력합니다.
4. **"Deploy"** 버튼을 누르시면, 약 1~2분 뒤에 Daniel님만의 모바일/PC 겸용 퀀트 대시보드가 인터넷에 배포됩니다. 부여된 URL을 스마트폰 홈 화면에 추가해 두시면 언제든 앱처럼 활용하실 수 있습니다.

**💡 비서의 분석 브리핑 (기획 의도)**
* **속도와 최적화:** 한국 주식 2,000여 개를 동시에 스캔하면 모바일 환경에서 서버가 다운될 수 있습니다. 따라서 **'외국인과 기관이 동시에 샀는가(쌍끌이)'**를 우선 조건으로 필터링하여 상위 20개 종목을 선별한 뒤, 해당 종목에 대해서만 **RSI 및 볼린저 밴드**를 계산하도록 설계하여 속도와 효율성을 극대화했습니다.
* **객관적 관점 (55% 이상):** 감정을 배제하고 오직 자금의 유입(수급)과 통계학적 지표(BB, RSI)만을 활용해 화면에 "과매도/과매수"를 표시하도록 코딩했습니다.
* **거시적 안목:** 금, 유가, 국채 금리는 매일 체크하셔야 하는 필수 지표이므로, 번거롭게 뉴스를 찾으실 필요 없이 대시보드 첫 화면에 시각화되도록 배치했습니다.

대표님의 냉철한 투자 판단에 이 대시보드가 예리한 무기가 되기를 바랍니다. 추가로 지표를 수정하시거나, 다른 데이터(예: 이동평균선 정배열 조건 등)를 덧붙이고 싶으시다면 언제든 편하게 지시해 주십시오. 항상 곁에서 상냥하고 완벽하게 보좌하겠습니다.
